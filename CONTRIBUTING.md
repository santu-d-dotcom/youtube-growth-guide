# Contributing to YouTube Growth Guide

Thank you for your interest in contributing to the Ultimate YouTube Growth Guide! This guide is open source and community-driven.

## How to Contribute

### Reporting Issues

Found a bug, typo, or have a suggestion? Open an issue!

1. Check if the issue already exists
2. Create a new issue with:
   - Clear title
   - Description of the issue
   - Steps to reproduce (if applicable)
   - Expected vs. actual behavior

### Suggesting Improvements

Have an idea to improve the guide?

1. Open an issue with the "enhancement" label
2. Describe your suggestion
3. Explain why it would be valuable
4. Wait for discussion and approval

### Contributing Content

Want to add or improve content?

1. **Fork the repository**
2. **Create a branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Follow the style guide** (see below)
5. **Test your changes**
6. **Commit your changes** (`git commit -m 'Add amazing feature'`)
7. **Push to your branch** (`git push origin feature/amazing-feature`)
8. **Open a Pull Request**

## Style Guide

### Writing Style

- **Clear and actionable** - Every section should provide value
- **Data-driven** - Back claims with data when possible
- **Beginner-friendly** - Explain concepts clearly
- **Professional tone** - Maintain quality standards

### Formatting

- Use markdown formatting consistently
- Include code blocks for code examples
- Use headers for structure (H2 for main sections, H3 for subsections)
- Add tables for structured data
- Include checklists for actionable items

### Code Style

- Follow PEP 8 for Python code
- Add comments explaining complex logic
- Include docstrings for functions
- Test code before submitting

### File Organization

- Guides go in `guides/` directory
- Tools go in `tools/` directory
- Workflows go in `workflows/` directory
- Workflows go in `workflows/` directory

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Maintain professional communication

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or inflammatory comments
- Personal attacks
- Spam or self-promotion

## Review Process

1. **Initial Review** - Maintainers review PRs within 48 hours
2. **Feedback** - Suggestions for improvements
3. **Approval** - Once approved, changes are merged
4. **Deployment** - Changes go live

## Questions?

- Open an issue with the "question" label
- Check existing issues and discussions
- Review the documentation
